# BirLiy Landing v2 — Design Spec

**Date:** 2026-05-26
**Author:** Claude (in session with Jack)
**Status:** Draft, pending Jack approval

---

## Goal

Заменить текущий лендинг BirLiy на v2, который:

1. **Точно отражает то, что реально работает в продукте** (`pos.ipaksavdo.uz` — 15 рабочих экранов, обойдено 26 мая 2026). Никаких выдуманных фич.
2. **Структурно полный как billz.uz** — есть proof, цены, segments grid, multi-column footer, FAQ глубже 5 вопросов. Сейчас лендинг выглядит как «маркетинговая брошюра», а не как «полноценный SaaS».
3. **Честная pre-launch рамка** — "early access" / "pilot cohort", без фейковых отзывов и без надувания "1000 клиентов".
4. **Сохраняет BirLiy editorial calm** (Sora + Manrope + paper/ink/green, hairlines, settle-in). Не превращаем в generic SaaS template.

---

## Architecture

- Single Next.js 14 App Router page (`app/page.tsx` → `components/LandingPage.tsx`).
- Решение: оставить single-file `LandingPage.tsx` (текущая конвенция) ИЛИ разнести на 16 секций-компонентов. **Рекомендация: разнести.** При 16 секциях single-file станет >1500 строк и плохо diff'ится. План: `components/landing/Section{Name}.tsx`, агрегатор в `LandingPage.tsx`.
- i18n: продолжаем RU primary + UZ mirror. Single `ru`/`uz` dict (как сейчас, typeof-locked).
- Source-of-visual-truth: `DESIGN.md` в корне (BirLiy editorial calm).
- Все настоящие фичи документированы в `.research/product-walk/` — 22 скрина из живого продукта.

---

## Tech Stack

- Next.js 14 App Router, TypeScript strict
- Tailwind 3 (с BirLiy токенами из `tailwind.config.ts`)
- `next/font/google` для Sora + Manrope + JetBrains Mono
- `framer-motion` для settle-in анимаций
- `lucide-react` для иконок (stroke-width 1.5)
- Без новых зависимостей: никаких Recharts/Swiper/Headless UI. Если нужны carousel/disclosure — пишем минимальные custom компоненты.

## Product Reality (для context, не на лендинг)

Что реально работает (из live walk):

| Категория | Фичи |
|---|---|
| **Касса** | split-pane касса + каталог, поиск по штрих-коду и названию, отложенный чек, привязка клиента к чеку |
| **Оплата** | 4 способа: Наличные, Терминал, QR-код, В долг. Калькулятор сдачи. Quick-add суммы (+10к/+50к/+100к) |
| **Каталог** | реестр SKU 9 064 позиций (национальная база), штрих-коды, ИКПУ (налоговый код), категории, бренды, импорт из Excel |
| **Склад** | приход, корректировка, импорт остатков, история движений, приём по сканеру |
| **Финансы смены** | касса дня с разбивкой по способам оплаты, журнал движений, открытие/закрытие смены, история смен, приход/расход |
| **Платежи** | полный реестр транзакций с фильтрами (статус, способ, дата, возвраты, поиск) |
| **Долги** | долговая книжка, экспорт Excel, регистрация клиентов |
| **Отчёты** | 5 KPI с дельтой %, hourly chart, разбивка по способам оплаты, топ-товары, продажи по кассирам |
| **Оборачиваемость** | классификация быстрые/медленные/мёртвые за 7/30/90 дней |
| **Клиенты** | база, фильтр должников, авто-наполнение из чеков |
| **Сотрудники** | роли (Владелец/Кассир/Суперадмин), PIN-доступ, последний логин, возраст PIN |
| **Audit log** | 12+ типов событий: создание товара, движения склада, платежи, открытие смены, онбординг |
| **Telegram бот** | уведомления о каждой продаже + команды `/today`, `/last 10` |
| **Настройки** | бизнес/точка, принтер чеков, тип бизнеса, смена PIN |
| **Возвраты** | через поиск чека |

Что **НЕ** работает (на лендинг не выносим):

- Фискальный чек (`/api/fiscal/.../receipt` → 404, страница «Загрузка чека...» висит). Это лечится продуктом.
- Мобильная навигация на 375px — сайдбар пропадает. Лендинг это не касается, продукт сам себя починит.

## Audience

**Primary buyer:** Собственник бизнеса (45-летний владелец продуктового магазина в районе Tashkent), не работал с программами кроме 1С. Хочет видеть «сколько денег в кассе сегодня» с телефона. Боится, что кассир ворует. Не верит, что «компьютер посчитает быстрее тетради».

**Daily user:** Кассир (20-35 лет, RU/UZ), работает на планшете или смартфоне в точке. Должен пробить чек за 15 секунд, не задерживая очередь.

**Pitch to:** Собственник. Кассир — это «удобство», но решение принимает владелец.

---

## Key Messages (top 5, from doc cross-synthesis)

1. **«Продажа за 15 секунд»** — canonical hook. Дословный демо-момент: scan → корзина → dynamic QR → оплата → электронный чек. Лидируем этим.
2. **«Касса отдельно, склад отдельно, банк отдельно»** — pain framing. Прямая цитата из docs. Собственник теряет деньги между системами, не видит картинку целиком.
3. **Free 6 месяцев → $7-$15/месяц** — низкий барьер входа. Cначала пробуй, потом плати. QR-комиссия 0.04%. Оборудование от $344 с финансированием.
4. **Локальная инфраструктура = доверие** — QR-оплата, интеграции и поддержка под рынок Узбекистана.
5. **«Для собственника, не только для кассира»** — отчёты, audit log, employee control. Решает главную тревогу владельца — «не вижу, что у меня в магазине». Это копи-якорь для всей секции отчётов.

## Visual Aesthetic (binding)

Read `DESIGN.md` first. Все правила оттуда. Кратко:

- **Палитра:** paper bg, ink текст, green только для action/success/win. ≤5 точек зелёного на экран.
- **Шрифты:** Sora display, Manrope body (поддерживает кириллицу), JetBrains Mono для табличных чисел.
- **Layout:** 12-колоночный grid на desktop. `section-shell` max 80rem. Section `py-24 lg:py-32`. Hairline `border-t border-mist` между секциями.
- **Motion:** `cubic-bezier(0.2, 0.8, 0.2, 1)`. Settle, не bounce.
- **Eyebrow numbering:** `01 / Section title` через все nameable секции (см. текущий LandingPage).
- **Никогда:** purple gradient, decorative blobs, 3-column icon grid с colored circles, glassmorphism, Inter/Roboto для display, bouncy motion.

---

## Voice & Framing Rules

- **Tone:** Plain, practical, unhurried. Short sentences. Real verbs. Никакого «экосистема», «synergy», «transformative», «innovative».
- **Pre-launch honesty:**
  - Нигде не говорим «1000+ магазинов используют BirLiy» — пилоты только идут.
  - Нет фейковых отзывов с именами клиентов. Вместо этого — секция «Ранний доступ» (см. структуру).
  - Trust strip говорит про exoсистему банка + IT Park статус + open pilot — это правда и достаточный proof.
- **Цены показываем открыто.** Не «свяжитесь с менеджером для расчёта». Тарифы публикуем.
- **Bugs не упоминаем.** Фискальный чек 404 — это техдолг продукта, на лендинге обещаем «электронный чек в Telegram», который работает.

## Structure (16 sections)

> Каждая секция: цель → contentplan → ключевая копия → визуальный тип → ссылки на real assets.

### 1. Header
- **Цель:** Навигация + единая точка action.
- **Content:** Logo (`birliy-wordmark.svg`) + nav (Возможности, Цена, FAQ) + LangPill (RU/UZ) + CTA «Получить ранний доступ» → `#lead`.
- **Pattern:** Текущий из `LandingPage.tsx`. Не меняем.
- **Eyebrow:** —

### 2. Hero
- **Цель:** За 3 секунды объяснить продукт + дать CTA + показать продукт работающим.
- **Headline:** «Продажа за 15 секунд» (Sora 80px desktop).
- **Subtitle:** «Касса, склад, QR-оплата и электронный чек — в одном приложении. Без компьютера, без интеграций, без вкладок.»
- **CTAs:** Primary «Получить ранний доступ» (green-500 pill, scroll-to-form `#lead`) + secondary «Посмотреть как работает» (text link, scroll-to `#how-it-works` — секция 15-Second Sale).
- **Trust line:** «Сделано для бизнеса Узбекистана · Open pilot 2026».
- **Right column (5/12):** HeroStatCard как сейчас, но с реальными цифрами из `/reports` (Выручка нал 21 000 UZS, Средний чек 7 000 UZS, Продаж 3) — берём из `.research/product-walk/10-reports.png`.
- **Layout:** 7/12 + 5/12 (как сейчас).

### 3. Trust strip (NEW)
- **Цель:** Honest pre-launch proof в один тонкий ряд под Hero, не перегружая.
- **Content:** 3 hairline-разделённых блока (text-only, без логотипов):
  - «Сделано для бизнеса Узбекистана»
  - «9 000+ товаров в базе» (реальная цифра из SKU реестра — `/products/catalog` показывает 9 064)
  - «Open pilot 2026» (честная фраза — мы запускаемся)
- **Visual:** Тонкий горизонтальный ряд, `text-xs uppercase tracking-wider text-ink-500`, hairline border сверху/снизу. Без иконок и логотипов — только текст.

### 4. The Pain (NEW)
- **Цель:** Resonance. Заставить владельца кивать. Доковая цитата дословно.
- **Eyebrow:** `01 / Знакомо?`
- **Headline:** «Касса отдельно. Склад отдельно. Банк отдельно. Вы — нигде вместе.»
- **Body:** «Кассир пробивает чек на одном устройстве. Остатки ведутся в тетради. Платежи приходят на терминал банка. Чтобы понять, сколько вы заработали сегодня — нужно сесть в офисе вечером и сводить три источника. К ночи вы устаёте и решаете «потом». Так месяц превращается в загадку.»
- **Visual:** Type-only. Никакой иллюстрации. Editorial cri-de-coeur.
- **Width:** `max-w-3xl` (узкая колонка, ощущается как обращение).

### 5. The 15-Second Sale (REWORK ProductMoment)
- **Цель:** Канонический демо-момент показать визуально. Это центральная сцена продукта.
- **Eyebrow:** `02 / Как проходит продажа`
- **Headline:** «Продажа за 15 секунд»
- **Body intro:** Один параграф: «Один экран. Шесть шагов. От сканирования товара до электронного чека покупателю — без переключений между приложениями.»
- **Visual:** 6-step horizontal sequence:
  1. Скан/выбор товара (mini mockup, из `16-kassa-empty.png` → `17-kassa-item-added.png`)
  2. Корзина (тот же, после добавления)
  3. К оплате — выбор способа (`18-payment-screen.png`)
  4. QR / наличные / терминал (`19-payment-cash.png`)
  5. Оплата прошла (`20-payment-success.png`)
  6. Чек в Telegram (concept mockup — Telegram message с чеком)
- **Pattern:** Каждый шаг = маленькая editorial карточка с номером, иконкой (lucide), коротким лейблом. Связываются тонкими стрелками. На mobile складываются вертикально.

### 6. Capabilities (REWORK — 6 cards)
- **Цель:** Полный feature catalog в одной секции, не растянутый.
- **Eyebrow:** `03 / Что внутри`
- **Headline:** «Шесть инструментов вместо шести приложений»
- **6 cards** (editorial, без colored backgrounds, just type + thin icon + metric line):
  1. **Многоканальная оплата** · 4 способа: наличные, терминал, QR, в долг. Калькулятор сдачи. _«Одна страница оплаты — четыре кнопки»_
  2. **Касса дня** · Смена в реальном времени: ожидаемая сумма, разбивка по способам, журнал движений. _«Сколько денег в кассе прямо сейчас»_
  3. **Каталог + склад** · 9 000+ товаров в базе. Штрих-коды, ИКПУ, импорт из Excel, приём по сканеру. _«Не заводить вручную»_
  4. **Отчёты с дельтой** · KPI с изменением к прошлому периоду, hourly chart, топ-товары, по кассирам. _«День, неделя, месяц — за один взгляд»_
  5. **Оборачиваемость** · Быстрые / медленные / мёртвые товары за 7/30/90 дней. _«Что заказать, что списать»_
  6. **Telegram уведомления** · Бот шлёт каждую продажу в Telegram. Команды `/today`, `/last 10`. _«Бизнес в кармане, без захода в приложение»_
- **Layout:** `md:grid-cols-2 lg:grid-cols-3`, gap 12. Hairline между rows. Каждая card = icon top + h3 title + body + italic metric line.

### 15. FAQ (EXPAND from 5 to ~10)
- **Цель:** Снять возражения. Покрыть top-вопросы потенциального клиента.
- **Eyebrow:** `09 / Вопросы и ответы`
- **10 questions (RU; UZ mirror):**
  1. Нужен ли компьютер? — Нет. Достаточно смартфона или планшета.
  2. Что с интернетом, если он плохой? — Продажи сохраняются локально и синхронизируются, когда связь восстановится. _(Note: feature claim — нужно подтвердить от продукта что offline-mode реально работает в текущей версии MVP.)_
  3. Сколько стоит? — 6 месяцев бесплатно, без условий. Платные тарифы появятся после периода раннего доступа — расскажем когда определим.
