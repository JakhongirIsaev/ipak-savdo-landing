- **JSON-LD разметка** — новый компонент `components/landing/JsonLd.tsx`, граф из 4 типов:
  - `Organization` (BirLiy) + `WebSite` + `SoftwareApplication` (категория BusinessApplication, оффер 49 000 UZS, в описании честно фигурируют ERP / biznes dasturi / POS) + `FAQPage`.
  - `FAQPage` собран из того же FAQ, что виден на странице (требование политики Google — разметка должна совпадать с видимым).

### Проверка и деплой
- `tsc` чистый, **148/148 тестов**.
- На живом проде JSON-LD валиден на обеих локалях; маршруты + `sitemap.xml` + `robots.txt` = 200.

### Честно про ожидания
- **Google почти игнорирует `<meta keywords>`** — но **Яндекс** (в UZ/RU) частично учитывает.
- Реально ранжироваться помогает: **JSON-LD + видимый контент + время** (недели, не дни).
- Сейчас ERP — только в «фоне», поэтому **в топ по «ERP» так не выйти** (для этого слово нужно в видимом тексте — отдельное решение на будущее).

## 3. Google Search Console

- Домен **`birliy.uz` подтверждён** (метод «Домен», через DNS) + `https://birliy.uz/` авто-подтверждён.
- `birliy.com` — случайная пустышка, висит «НЕ ПОДТВЕРЖДЕНО», ничему не мешает (можно удалить).
- Метод DNS = **код/деплой не нужны**, Google-часть закрыта.

**Осталось сделать Jack:**
- Отправить карту сайта: ресурс `https://birliy.uz/` → «Файлы Sitemap» → вписать `sitemap.xml` → «Отправить».
- Через пару недель в **«Эффективность» → «Запросы»** будет видно, по каким небрендовым словам находят сайт.

## 4. Как находят сайт «не по слову birliy» (небрендовый трафик)

- **Брендовый поиск** («birliy») = кто уже знает. **Категорийный** («kassa dasturi», «ERP tizimi», «POS Toshkent») = новые клиенты.
- Чтобы появляться по категорийным запросам нужно: **слово в видимом тексте + отдельные страницы**, **ссылки с других сайтов**, **время**. По «горячим» словам уже сидят конкуренты (Billz, Smartup, 1C, Jowi).
- **Где это видно:** GSC → «Эффективность» → «Запросы».
- **Быстрые каналы** (не ждать месяцами): **Google Ads** (платный поиск — сразу в топе), **Instagram/Telegram**, **Google Business / каталоги**.

## 5. Instagram — план автопостинга

**Решения Jack:**
- Движок = **готовый планировщик (Metricool)**, не свой бот.
- Контроль = **аппрув в Telegram** (каждый пост сначала на ✅), не полный автомат.

**Как будет работать:** я готовлю пост (RU+UZ + картинка) → показываю на аппрув → одобренное ставлю в Metricool → он сам публикует. **Jack Instagram не открывает.**

**Честные ограничения:**
- Я **не могу** постить в IG напрямую из чата (нет коннектора; вход паролем / browser-automation = риск бана). Безопасно — только официальный **Meta Graph API**, который и используют планировщики.
- «Тапнул в Telegram → улетело» в чистом виде — это свой бот (не выбран). С готовым планировщиком аппрув = «показал → одобрил → поставил в расписание».

**Незыблемый prereq (только Jack, телефон, ~10 мин):**
1. Instagram BirLiy → перейти на **бизнес-аккаунт**.
2. Создать **страницу в Facebook** и **связать** с этим Instagram.
   *(Meta даёт автопостинг только через связку бизнес-IG ↔ FB-страница. Обходов нет.)*

**Контент:** заготовки уже есть в `marketing/` (12 IG-постов + 10 TG, RU+UZ). Картинки — через Higgsfield (тратит кредиты).
