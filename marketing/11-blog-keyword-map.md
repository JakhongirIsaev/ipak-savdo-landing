# Blog keyword-to-page map (anti-cannibalization)

Source: `D:\birliy-seo-content\` (clusters cluster-uz-magazin-uchun-dastur, cluster-ru-programma-dlya-magazina; briefs magazin-uchun-dastur, programma-dlya-magazina, qarz-daftar, dolgovaya-tetrad, dokonda-nima-qolgan).

**Rule:** exactly ONE canonical target page per primary keyword. A page may rank for many long-tail variants, but two pages must never target the same primary keyword (that is cannibalization). When a live post already targets a keyword, **upgrade it**; do not create a second page for the same term.

Legend: **LIVE-UPGRADE** = deepen an existing post (no new URL). **LIVE-KEEP** = already owns a distinct keyword, leave as is. **NEW** = build a new post. **GATED** = build only with verified official sources.

## 3. Cannibalization guardrails (the pairs most at risk)

1. **Head term is owned, once.** Only `magazin-uchun-dastur-telefonda-savdo` may target the bare "magazin uchun dastur" / "программа для магазина". Every new page must use a **segment- or feature-qualified** primary keyword (minimarket uchun kassa dasturi, программа для минимаркета, программа для магазина у дома, kassa apparatsiz savdo), never the bare head term in H1/title/slug.
2. **"kassa dasturi" (NEW minimarket page) != "POS tizimi" (live pos-tizimi).** Different primary keyword, different H1. If GSC later shows them competing for the same queries, MERGE the new page into `pos-tizimi` rather than keep both.
3. **"программа для минимаркета" (NEW ru) != "POS-система для минимаркета" (live pos-tizimi ru).** Same rule.
4. **One qarz page.** `qarz-daftar-orniga-nima` owns qarz daftar / nasiya / долговая тетрадь. Do not spin up a second nasiya/долги article (cluster C2 "nasiya savdo") unless GSC shows a clearly distinct, uncovered query.
5. **One stock page.** `dokonda-nima-qolganini-telefondan-bilish` owns "nima qolgan / что осталось". A broader "ombor hisobi telefonda" spoke (cluster D1) must take the management angle with a distinct keyword, or be dropped (the live `skladskoy-uchet` already covers "складской учёт").

## 4. Internal-link skeleton (apply when writing)

- Pillar (`magazin-uchun-dastur-telefonda-savdo`) links DOWN to every spoke; every spoke links UP to the pillar. Anchor = the target keyword, never "bu yerga"/"click here".
- New P1 spokes interlink with their nearest sibling (kassa-apparatsiz <-> minimarket-kassa-dasturi <-> narx; programma-u-doma <-> programma-dlya-minimarketa).
- Each existing LIVE-KEEP post gets ONE link up to the pillar on its next edit (closes orphan risk).
- Every new RU page ships with a UZ sibling (and vice versa) wired as hreflang alternates (`/blog/...` uz <-> `/ru/blog/...` ru <-> `/en/blog/...` en).
